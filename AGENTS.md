# AGENTS.md

本仓库用于维护本地可复用的 agent skills。协作者进入仓库后，先把这里当作“技能内容仓库”而不是普通应用项目。

## 目标

- 在 `skills/` 下维护结构清晰、可复用、可审阅的 skill
- 尽量把操作流程、依赖、脚本入口写进 `SKILL.md`
- 避免把本地环境、密钥、临时产物提交进 Git

## 仓库结构

```text
skills/
  <skill-name>/
    SKILL.md
    references/
    scripts/
    tests/
    agents/
```

说明：

- `SKILL.md`：skill 的主入口，必须说明用途、触发条件、前置要求、执行流程
- `references/`：补充文档，只放被 `SKILL.md` 明确引用的内容
- `scripts/`：可执行脚本、依赖清单、示例配置
- `tests/`：和脚本或关键流程对应的测试
- `agents/`：该 skill 需要的额外 agent 配置

## 新增 Skill 约定

新增 skill 时至少补齐：

1. `skills/<name>/SKILL.md`
2. 必要的 `references/` 或 `scripts/`
3. 若存在可执行逻辑，补最小可运行验证方式

建议 `SKILL.md` 至少包含这些部分：

- 名称与用途
- 何时使用 / 不何时使用
- 前置依赖
- 标准工作流
- 快速命令示例
- 常见错误或风险
- 参考文件路径

## 修改原则

- 优先小步修改，不做大范围无关重写
- 更新脚本时，同步检查 `SKILL.md` 是否仍然准确
- 新增命令示例时，不要写入真实密钥、Token、内网地址
- 若 skill 自带脚本入口，优先复用脚本，不在文档里堆大量重复命令

## 提交前检查

- `SKILL.md` 中的路径是否真实存在
- 示例命令是否与当前脚本参数一致
- `.env`、`.venv`、缓存、截图、标注图等本地产物未被提交
- 若新增脚本，至少说明运行方式与依赖安装方式

## 不要做的事

- 不要提交密钥、Token、Cookie、会话文件
- 不要把临时调试输出留在仓库根目录
- 不要在没有必要时引入大型二进制或虚拟环境目录
- 不要修改与当前任务无关的 skill
